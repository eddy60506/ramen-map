## 編輯店家資料（給編輯者看）

所有店家資料都在 **`data.js`** 這個檔案裡，是一個列表。每家店都長這樣：

```js
{
    "name": "金色不如歸 拉麵",
    "rank": "TBD",
    "area": "信義",
    "address": "台北市信義區忠孝東路四段553巷22號",
    "soup": "鹽味",
    "price": "380 - 520 新台幣",
    "lat": 25.04199,
    "lng": 121.56243,
    "plusCode": ""
}
```

### 欄位說明

| 欄位 | 必填 | 說明 | 範例 |
|------|------|------|------|
| `name` | ✅ | 店名（吃過後可加上分店名稱） | `"麵屋一燈 台北本店"` |
| `rank` | ✅ | 評級，**只能填**：`T0` `T1` `T2` `T3` `T4` `T5` `T6` `TBD` | `"T1"` |
| `area` | ✅ | 行政區，**不要加「區」字** | `"信義"` `"中山"` `"大安"` |
| `address` | ✅ | 完整地址 | `"台北市信義區松壽路9號"` |
| `soup` | ✅ | 湯頭 | `"豚骨"` `"醬油"` `"雞白湯"` `"沾麵"` |
| `price` | ✅ | 價格範圍 | `"260 - 380 新台幣"` |
| `lat` | ✅ | 緯度（小數點 5 位即可） | `25.04199` |
| `lng` | ✅ | 經度 | `121.56243` |
| `plusCode` | ⬜ | 不會就留 `""` | `""` |

### 格式規則（很重要！）

1. **每個欄位用逗號 `,` 分開**，最後一個欄位不要逗號
2. **每家店用大括號 `{ }` 包起來**，店與店之間用逗號分開
3. **文字一律用雙引號 `"`**（不是單引號 `'`，也不是中文「」）
4. **數字不要加引號**：`"lat": 25.04199` ✅ ／ `"lat": "25.04199"` ❌
5. 每次儲存後，到網站硬重新整理（`Cmd+Shift+R`）確認沒壞掉

### 怎麼找 lat / lng（經緯度）

1. 用 Google Maps 搜店名
2. 在地圖上**右鍵點店家位置**
3. 跳出選單的**第一行**就是兩個數字（例：`25.041996, 121.562431`）
4. **左邊的是 lat，右邊的是 lng**，貼到對應欄位

### 新增一家店的流程

1. 開 `data.js`
2. 找到列表結尾的 `]`
3. 在最後一筆店家後面加逗號 `,`，貼上完整一格 `{ ... }`
4. 存檔 → push → 等 30 秒 → 硬重新整理網站

### 評過的店家：把 `rank` 從 `TBD` 改成實際等級

例如吃過麵屋一燈覺得超讚：

```js
"rank": "TBD",   // 改之前
"rank": "T0",    // 改之後
```

## 協作建議（最簡單）

**推薦做法：在 GitHub 網頁上直接編輯，不用裝任何東西。**

### 編輯者的設定（一次性，5 分鐘）

1. 註冊 GitHub 帳號：<https://github.com/signup>
2. 把 GitHub 帳號名告訴專案 owner（Eddy）
3. Owner 把你加為 Collaborator：
   - Repo → Settings → Collaborators → Add people → 輸入帳號
4. 編輯者 email 收到邀請信 → 點 Accept

### 編輯流程（每次新增/修改店家）

1. 打開 <https://github.com/eddy60506/ramen-map/blob/main/data.js>
2. 點右上角的**鉛筆圖示** ✏️（Edit this file）
3. 直接修改內容
4. 拉到頁面最下面：
   - **Commit message** 寫一句話描述（例：「新增 一蘭 西門店」）
   - 選 **Commit directly to the main branch**
   - 按綠色按鈕 **Commit changes**
5. 等 30~60 秒 → 網站自動更新

### 為什麼推這個方式

| 方式 | 難度 | 即時生效 | 推薦 |
|------|------|----------|------|
| **GitHub 網頁編輯** | ⭐ | ✅ | ✅ 最推薦 |
| 本機 git clone + 編輯 | ⭐⭐⭐ | 需 push | 給工程師 |
| 用 Google Sheets 同步 | ⭐⭐⭐⭐ | 需設 sync | 太複雜 |
| 弄個後台管理介面 | ⭐⭐⭐⭐⭐ | 即時 | 殺雞用牛刀 |

GitHub 網頁編輯的優點：
- 不用裝 Git、不用裝編輯器
- 改錯了 GitHub 會在 commit history 留紀錄，可隨時還原
- 多人同時編輯，GitHub 會處理衝突
- 全程瀏覽器內完成
